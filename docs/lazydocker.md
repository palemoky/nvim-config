# LazyDocker

Lazydocker 与 Lazygit 是同一个作者，在实现原理方面，lazydocker 并不是直接调用 cli 命令，而是直接调用 Docker API，以获得更好的性能。

> 以下内容直接在界面按 ? 即可获取

## 界面与导航

| 按键             | 功能                                      |
| ---------------- | ----------------------------------------- |
| Tab/左右按键/1~6 | 切换面板                                  |
| `[`/`]`              | Log 的标签页切换                          |
| Enter            | 聚焦到右侧面板                            |
| `+`/`-`              | 切换面板的宽度（半屏/全屏），看日志很有用 |
| `/`                | 搜索                                      |

## 容器操作

| 按键 | 功能                                                         |
| ---- | ------------------------------------------------------------ |
| `m`    | 快速打开并实时流式阅读选定容器的日志（等同于 docker logs -f），Ctrl+c 返回 lazydocker。复制日志很有用 |
| `E`    | 直接进入容器终端                                             |
| `s`    | 停止容器                                                     |
| `p`    | 暂停容器                                                     |
| `r`    | 重启容器                                                     |
| `d`    | 删除容器                                                     |
| `a`    | 直接附加到容器的输入输出流上                                 |
| `e`    | 显示/隐藏已停止的容器                                        |

## Compose 操作

在含有 `docker-compose.yml` 的目录下启动 lazydocker，它会自动以服务组的形式展示

| 按键 | 功能                                                         |
| ---- | ------------------------------------------------------------ |
| `u`    | Up service（单独拉起/启动选中的服务）                        |
| `d`    | 删除该服务的容器                                             |
| `U`    | Up project（相当于 `docker compose up -d`，拉起整个项目）    |
| `D`    | Down project（相当于 docker compose down，关闭并清理整套服务） |

## 镜像、卷

| 按键 | 功能                     |
| ---- | ------------------------ |
| `d`    | 退出                     |
| `b`    | 批量清理无用镜像、数据卷 |
